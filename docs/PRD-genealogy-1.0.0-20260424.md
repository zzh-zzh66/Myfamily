## 1. 产品概述

### 1.1 产品定位

| 项目 | 内容 |
|------|------|
| 产品名称 | MyFamily |
| 核心定位 | 打破家族成员隔阂，增强家族凝聚力 |
| 核心价值 | 无壁垒交流、族谱可视化、情感传承 |
| 目标用户 | 中小型、大型家族的普通成员和管理员 |
| 设计风格 | 中国风水墨画 |

### 1.2 核心价值主张

1. **打破交流壁垒**：无需添加好友、无需互相关注，直接查看家族成员信息并发送邮件
2. **族谱可视化**：水墨风格族谱树，传承家族文化
3. **零门槛参与**：任何家族成员都可以浏览、发帖、参与家族事务

## 6. 数据库设计要点

### 6.1 核心表结构

| 表名 | 说明 |
|------|------|
| t_family | 家族表 |
| t_member | 成员表 |
| t_member_profile | 成员详情表 |
| t_genealogy_tree | 族谱关系表 |
| t_mail | 邮件表 |
| t_post | 动态表 |
| t_comment | 评论表 |
| t_memorial | 纪念堂表 |
| t_user | 用户表 |

### 6.2 族谱关系

- 使用邻接表模型（Adjacency List）
- father_id 维护父系关系
- generation 字段存储辈分信息
