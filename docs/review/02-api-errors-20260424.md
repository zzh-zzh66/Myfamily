# MyFamily 项目问题总结

**日期**: 2026-04-24
**项目**: MyFamily 族谱管理系统
**版本**: v1.0.0

## 一、本次解决的问题

### 1.1 系统繁忙错误 (API 调用失败)

**错误信息**:
```
java.lang.IllegalArgumentException: Name for argument of type [java.lang.Long] not specified, and parameter name information not found in class file either.
```

**根本原因**: `@RequestParam` 注解缺少显式的 `value` 属性，Spring 无法获取参数名称

**受影响的文件**:
- `MailController.java` - folder, page, size 参数
- `MemberController.java` - familyId, page, size 参数
- `PostController.java` - familyId, type, page, size 参数

**修复方案**: 为所有 `@RequestParam` 注解添加显式的 `value` 属性

```java
// 修复前
@RequestParam(defaultValue = "1") int page

// 修复后
@RequestParam(value = "page", defaultValue = "1") int page
```

---

### 1.2 MailService NullPointerException

**错误信息**: 当用户 `member_id` 为 NULL 时，调用 `user.getMemberId()` 导致 NPE

**受影响的文件**: `MailService.java`

**修复方案**: 添加 null 检查

```java
Long memberId = user != null ? user.getMemberId() : null;
if (memberId == null) {
    throw new BusinessException("用户未关联成员信息");
}
```

**修复位置**:
- `getMailList()` 方法 (第 51-53 行)
- `deleteMail()` 方法 (第 148-151 行)
- `getUnreadCount()` 方法 (第 162-164 行)

---

### 1.3 CommentService NullPointerException

**错误信息**: `deleteComment()` 方法中 `user` 可能为 null

**受影响的文件**: `CommentService.java` 第 65 行

**修复方案**:
```java
// 修复前
if (!"ADMIN".equals(user.getRole()))

// 修复后
if (user == null || !"ADMIN".equals(user.getRole()))
```

---

### 1.4 PostService NullPointerException

**错误信息**: `deletePost()` 方法中 `user` 可能为 null

**受影响的文件**: `PostService.java` 第 139 行

**修复方案**:
```java
// 修复前
if (!post.getAuthorId().equals(userId) && !Constants.Role.ADMIN.equals(user.getRole()))

// 修复后
if (!post.getAuthorId().equals(userId) && (user == null || !Constants.Role.ADMIN.equals(user.getRole())))
```

## 二、历史问题回顾

### 2.1 登录失败 - BCrypt 密码哈希重复

**问题**: 数据库中密码哈希前缀 `$2a` 重复出现

**原因**: 密码哈希生成时处理不当

**修复方案**: 使用正确的 BCrypt 哈希值覆盖

```sql
UPDATE t_user SET password_hash = '$2a$10$...正确哈希值...' WHERE username = 'xxx';
```

---

### 2.2 JWT 认证过滤器包名错误

**问题**: `JwtAuthenticationFilter` 放置在 `config` 包下，但 Spring Security 配置导入时路径错误

**修复**: 移动到 `filter` 包并修正 import 语句

---

### 2.3 缺失的端点

**问题**: 前端调用 `/genealogy/tree` 但后端未实现该端点

**修复**: 在 `MemberController` 中添加 `/api/v1/members/genealogy/tree` 端点

---

### 2.4 前端 API 路径错误

**问题**: 前端 API 路径与后端不匹配

**修复**:
```typescript
// 修复前
return request.get('/genealogy/tree', {...})

// 修复后
return request.get('/members/genealogy/tree', {...})
```

---

## 三、代码质量改进

### 3.1 Maven 编译配置

**添加了 `-parameters` 编译器参数**:
```xml
<maven.compiler.parameters>true</maven.compiler.parameters>
```

**显式配置 maven-compiler-plugin**:
```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-compiler-plugin</artifactId>
    <configuration>
        <parameters>true</parameters>
    </configuration>
</plugin>
```
