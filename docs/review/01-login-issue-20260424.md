## 1. 问题概述

### 1.1 问题描述

前端用户无法使用初始账号登录系统，但通过注册接口创建的新账号可以正常登录。

### 1.2 影响范围

- 初始账号：admin、li_zhiqiang、li_xiaoyan、li_jianjun、li_jianfang
- 无法登录，但数据库中密码哈希看起来正确

### 1.3 问题分类

- **类型**：密码验证失败
- **优先级**：P0（阻塞问题）

---

## 2. 问题排查过程

### 2.1 第一阶段：检查数据库

| 步骤 | 操作 | 结果 |
|------|------|------|
| 1 | 检查数据库中密码哈希 | 哈希格式正确，长度60字符 |
| 2 | 验证BCrypt哈希值 | 从 test 用户复制的哈希可以登录 |
| 3 | 发现问题 | admin 等账号的哈希前缀被重复添加 |

### 2.2 第二阶段：检查后端日志

| 步骤 | 日志信息 | 分析 |
|------|----------|------|
| 1 | `Encoded password does not look like BCrypt` | 后端认为密码哈希格式不正确 |
| 2 | `password: \.20cQQubK3...` | 开头 `$2a$10$` 丢失，变成 `\ .20c...` |
| 3 | 分析HEX值 | 确认是 `$` 符号在某个环节被丢失 |

### 2.3 第三阶段：定位根本原因

| 环节 | 问题描述 |
|------|----------|
| 数据库连接 | 字符集正确（utf8mb4_unicode_ci） |
| init_data.sql | BCrypt哈希本身正确 |
| 手动UPDATE | PowerShell命令中 `$` 符号被转义/吃掉 |
| SQL客户端更新 | 第一次更新受影响的行:0（已正确），但后续操作可能覆盖 |

### 2.4 最终发现

**根本原因**：用户在手动更新密码时，不小心添加了重复的BCrypt前缀：
- 错误：`$2a$2a$10$aAsLCGciRdk/...`
- 正确：`$2a$10$aAsLCGciRdk/...`

## 3. 解决方案

### 3.1 方案对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| 方案1：复制test用户哈希 | 简单，不影响数据 | 需要知道test用户的密码 |
| 方案2：删除旧账号重建 | 彻底 | 关联数据需要处理外键 |
| 方案3：通过TestController生成 | 正确 | 需要重启后端 |

### 3.2 最终采用方案

**采用方案1**：复制能登录的 test 用户的密码哈希给其他账号

```sql
-- 确认test用户的正确哈希
SELECT password FROM t_user WHERE username = 'test';
-- 结果：$2a$10$aAsLCGciRdk/yXTu3DI6Pu0Vg6kmliJct4ux3MRX3UTUA6TgFVPaO

-- 更新其他账号
UPDATE t_user SET password = '$2a$10$aAsLCGciRdk/yXTu3DI6Pu0Vg6kmliJct4ux3MRX3UTUA6TgFVPaO'
WHERE username IN ('admin', 'li_zhiqiang', 'li_xiaoyan', 'li_jianjun', 'li_jianfang');
```

## 4. 经验教训

### 4.1 技术层面

| 问题 | 教训 | 改进措施 |
|------|------|----------|
| BCrypt哈希格式 | BCrypt哈希以 `$2a$10$` 开头，60字符长度 | 在init_data.sql中标注清楚 |
| PowerShell中$符号 | PowerShell会把 `$` 视为特殊字符 | 使用SQL文件而非命令行 |
| 外键约束 | 删除用户前需要处理关联数据 | 添加禁用外键检查或先更新关联字段 |

### 4.2 流程层面

| 问题 | 教训 | 改进措施 |
|------|------|----------|
| 密码验证 | 数据库查询结果≠后端实际读取 | 需要验证后端日志 |
| 测试不充分 | 初始数据未经完整流程测试 | 添加集成测试 |

## 7. 附录

### 7.1 正确的BCrypt哈希格式

```
$2a$10$dXJ3SW6G7P50lGmMkkmwe.20cQQubK3.HZWzG3YB1tlRy.fqvM/BG
│  │ │──────────────────────────────────────────────────────────────│
│  │                        53字符的salt+hash                       │
│  │ salt的work factor (10 rounds)                                 │
│BCrypt算法标识                                                     
```

### 7.2 验证命令

```sql
-- 验证哈希格式正确（应为60字符）
SELECT LENGTH(password) FROM t_user WHERE username = 'admin';

-- 验证HEX值正确
SELECT HEX(password) FROM t_user WHERE username = 'admin';
-- 正确结果应以 243261 开头（$2a 的十六进制）
```

### 7.3 测试接口

| 接口 | 说明 |
|------|------|
| `GET /api/v1/test/bcrypt?password=xxx` | 生成BCrypt哈希 |
| `GET /api/v1/test/bcrypt/verify?raw=xxx&encoded=yyy` | 验证密码是否匹配 |
